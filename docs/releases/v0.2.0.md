# MiroFish World Fork v0.2.0

This release marks the first clearly documented release of the world-simulation fork line.

## 中文

### 这版是什么

这不是一次零散 patch 的集合，而是把这条 fork 线真正收束成一个可运行、可观测、可续跑、可评估的 `world mode` 版本。

### 核心变化

- 把 `world mode` 提升为一等运行时
- 补齐 `checkpoint -> resume -> restore -> finalize` 的链路
- 打通 operator CLI：
  - `world_run.py run`
  - `resume`
  - `status`
  - `restore`
  - `finalize`
  - `staged`
- 打通运行态观测：
  - `run_state.json`
  - `checkpoint.json`
  - `actions.jsonl`
  - `state_snapshots.jsonl`
  - diagnostics / reports
- 支持多 provider / 多 model 路由：
  - `llm_registry.json`
  - OpenClaw 复用
  - world actor 级 `llm_selector`
- 沉淀 preset / eval / case set：
  - `world_runtime_presets.json`
  - `world_model_eval_*`
  - `world_case_sets/*`

### 这版解决的关键问题

- operator 跑 world 时，服务/UI 侧之前可能拿不到正确的 `run_state`
- 某些历史 world run 已经完成，但 `run_state` 与 checkpoint 不一致
- world 续跑、阶段运行、诊断导出虽然存在，但没有真正形成稳定工作流
- 文档叙事仍停留在 upstream 的“通用预测引擎”，没有反映这条 fork 实际交付的能力

这一版之后：

- operator 路径中途可看，终态可对账
- 历史 world run 可以从 artifacts 自动修复状态
- 真实 world case 已经可以长期推进，并通过 diagnostics / eval 持续复盘

## English

### What this release is

This is the first release that turns the world-simulation fork into a clearly documented, operationally coherent version rather than a pile of local patches.

### Core changes

- `world mode` is now treated as a first-class runtime
- the `checkpoint -> resume -> restore -> finalize` chain is hardened
- the operator CLI is now part of the real workflow:
  - `world_run.py run`
  - `resume`
  - `status`
  - `restore`
  - `finalize`
  - `staged`
- runtime observability is now explicit:
  - `run_state.json`
  - `checkpoint.json`
  - `actions.jsonl`
  - `state_snapshots.jsonl`
  - diagnostics / reports
- multi-provider / multi-model routing is supported through:
  - `llm_registry.json`
  - OpenClaw reuse
  - per-actor `llm_selector`
- presets, evals, and durable case sets are now part of the repository

### What this release fixes in practice

- operator-launched world runs can now be observed correctly by the service / UI side
- stale or missing world run state can now be rebuilt from runtime artifacts
- staged runs, resume, and diagnostics now form a real working operator flow
- the repository docs now match the actual delivered capabilities of this fork
